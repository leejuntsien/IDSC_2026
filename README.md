## Machine Learning & Deep Learning Pipeline
This repository contains a full set of machine learning and deep learning scripts optimized for ECG and VCG anomaly extraction (such as diagnosing Brugada syndrome vs Normal patients). All core logic is inside the `ml_pipeline/` module, with dedicated runner scripts located in the root.

### 1. `ml_pipeline/data_loader.py`
This script serves as the bridge between raw PhysioNet WFDB `.dat` files and our ML models. 
- **`load_wfdb_record`**: Parses raw signal DataFrames directly from the downloaded dataset without needing intermediate storage.
- **`extract_discrete_features`**: Wraps the `ecg_pipeline_features` script to identify R-peaks, delineate waveforms (P, Q, R, S, T), and fetch tabular, beat-by-beat physiological traits (QRS duration, Voltage differences, etc.).
- **`extract_sequence_features`**: Time-series sequential extractor. It can parse individual leads, entire 12-lead setups (`use_all_leads=True`), or Kors-transformed 3D VCG signals (`use_vcg=True`). It unifies variable RR intervals using either zero-padding or `scipy.signal` interpolation (resampling) to explicitly set sequence array shapes.

### 2. `ml_pipeline/classic_pipeline.py`
Provides an automated Scikit-Learn evaluation wrapper for the discrete tabular features.
- Contains hyperparameter search pipelines for `RandomForest`, `SVC`, unsupervised anomaly detection via `IsolationForest`, `LightGBM`, and `XGBoost`.
- Includes a robust median imputer (`SimpleImputer`) to patch missing physiological features (such as U-waves missing in certain subjects).
- Conducts `StratifiedKFold` Grid Searches minimizing for `f1_macro` disparity (handling class imbalances naturally).
- Computes advanced interpretability and performance metrics: **Brier Scores**, **ROC Area-Under-Curve**, **Precision-Recall AUC**, **SHAP** Value tree-explainers (or Top-K Bar Plots), and plots `learning_curve` cross-validation tracks.

### 3. `ml_pipeline/dl_pipeline.py`
PyTorch-centric sequential architectures strictly evaluating the continuous time-series blocks generated by the DataLoader.
- Contains the custom PyTorch `ECGSequenceDataset` wrapper.
- **`ECGCNN1D`**: A 1D-Convolutional Neural Network capable of adapting its channels to single-lead, 3-lead VCG, or full 12-channel inputs dynamically.
- **`ECGTransformer`**: A classic self-attention sequence architecture (TransformerEncoder) for temporal sequence pooling.
- **`ECGAttentionLSTM`**: A Bidirectional LSTM containing an intrinsic self-attention pooling mechanism.
- Includes transparency mapping functions (`plot_attention_overlay` and `plot_saliency_overlay`) to mathematically isolate problematic segments in the waveform directly mapping to gradients highlighting where the model detected the anomaly.

## How to Execute the Pipelines

### Classic ML Execution
Run `python run_brugada_classic_ml.py`.
This loop explicitly calls extraction on all records, drops metadata IDs, performs the 70/30 `train_test_split`, and passes the pure tabular features into the Tuning Pipeline (e.g. LightGBM / Random Forest).
* **Interpretability:** Generates a side-by-side subplot tracking a cleanly classified normal heartbeat against an actual Brugada segment flagged by the model within the test set.

### Deep Learning Execution
Run `python run_brugada_explainability.py`.
This loop generates 12-lead PyTorch tensors, trains the CNN 1D classifier over the waveforms via Binary Cross Entropy loss, and outputs pure transparency diagnostics.
* **Explainability:** Computes an Automatic Differentiation backwards-pass mapping gradients indicating which numerical points explicitly altered the diagnosis. It then triggers `plot_saliency_overlay` to color the raw heartbeat background marking exactly where the Brugada characteristic fired.
